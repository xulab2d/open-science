# Lab Conventions

General conventions inferred from legacy materials:
- Favor human project names plus sample/run context over opaque filesystem labels.
- Separate evidence from interpretation.
- Use project-specific scripts and established observables when they exist.
- Prefer concise, science-forward summaries over operational trivia.

Common data-analysis cautions:
- Gate-to-`(n, D)` calibration is project-specific and should be verified from the project’s own analysis pipeline.
- Broad-window peak or extrema trackers can hop between features; continuity checks or fits are often safer.
- Instrument offsets and background subtraction can look like physics if detached from project context.

Communication norms worth preserving:
- Ask targeted questions only when they unblock interpretation or routing.
- Route project-specific questions to project owners; route tooling questions to the lab-assistant maintainer.
- Keep external summaries brief and useful.
