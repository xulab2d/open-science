# Keep / Rewrite / Drop

Keep:
- provenance and uncertainty discipline
- project and people memory
- concise science-first summaries
- targeted clarification behavior
- recurring project list

Rewrite:
- all core instructions into a smaller Codex-native set
- project and convention notes into concise canonical files
- Slack plans as an integration layer instead of a global behavior source
- any reusable automation as small deterministic tools

Drop:
- OpenClaw agent/session architecture
- Dropbox cursor-sync as the default operating model
- heartbeat/cron orchestration as the assistant’s core identity
- redundant prompt files with overlapping authority
- large generated output trees as part of the new workspace design

Defer:
- active-run monitoring code
- corpus/anomaly machinery
- any experiment-specific automation that has not yet proven durable value in the Codex workflow
