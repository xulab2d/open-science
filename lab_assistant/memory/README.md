# Memory

Purpose:
- Store lightweight durable notes that do not belong in the core directives or project canon.
- Keep a compact record of what is working, what is failing, and what still needs promotion into canon.

Rules:
- Prefer updating `context/` first.
- Use `notes/` for dated or transitional notes.
- Periodically compact repeated notes into canonical files.
- Use `review_log.md` for short success/failure entries that should feed future maintenance.

Not for:
- Raw data outputs
- Long operational logs
- Framework-specific state
