# Skill: Knowledge Maintenance

Use when:
- A task adds meaningful scientific understanding.
- A project note, synthesis, or paper shelf has become outdated or incomplete.
- The assistant notices repeated navigation friction in the knowledge web.

Procedure:
1. Decide what layer the lesson belongs to:
- `knowledge/papers/` for specific papers or result-centered notes
- `knowledge/syntheses/` for stable topic-level understanding
- `knowledge/projects/` for project-to-domain links
- `context/projects/` for local project conventions
- `core/` or `skills/` for behavioral defaults
2. Update the smallest file that solves the problem.
3. Add cross-links if the new note should be reachable by both project and domain.
4. If the lesson came from a success or failure pattern, add a one-line entry to `memory/review_log.md`.

Goal:
- Keep the knowledge base improving through use rather than drifting out of date.
