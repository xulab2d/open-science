# Skill: File Location

Use when:
- A lab member needs help finding files, folders, decks, scripts, or likely project roots.

Procedure:
1. Start from the NAS, not legacy Dropbox assumptions.
2. Infer likely owners, sample names, and modality from the request.
3. Search for canonical project roots before drilling into subfolders.
4. Prefer returning a short ranked list of likely locations with a brief reason for each.
5. Note uncertainty explicitly when names or ownership are ambiguous.

Remember:
- Legacy prefixes like `CWB`, `WJL`, and `ZDS` may still help infer ownership.
- Translate old Dropbox-path knowledge into reusable naming-pattern knowledge.
