# Skill: Capture Feedback

Use when:
- A lab member corrects a summary, plot, routing choice, or interpretation preference.
- A task reveals a repeated success pattern that should become default behavior.

Procedure:
1. Decide whether the feedback is:
- global preference
- project-specific convention
- person-specific preference
- one-off note
 - reusable success pattern
2. Record it immediately in the smallest canonical file that fits.
3. If it changes default behavior, update `core/preferences.md` or a skill.
4. If it only affects one project, update that project note instead.
5. If it exposes a gap, miss, or recurring friction point, append a short entry to `memory/review_log.md`.
6. If the lesson improves scientific recall, update `knowledge/syntheses/` or `knowledge/papers/`.

Goal:
- The same mistake should not need to be corrected twice.
- Repeated wins should become easier to reproduce.
