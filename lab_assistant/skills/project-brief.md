# Skill: Project Brief

Use when:
- The assistant needs to create or refresh a concise project summary.

Procedure:
1. Identify owners, sample/run names, modalities, and canonical NAS roots.
2. Capture only durable information:
- what the project is
- where files live
- which conventions matter
- what questions remain open
3. Prefer one canonical project note over many dated fragments.
4. Link to important scripts, decks, or summaries only if they are stable references.

Avoid:
- Long chronological logs.
- Repeating operational details that will age quickly.
